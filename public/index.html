API Testing Guide for Version 1.0.0

This guide provides instructions on how to test various API endpoints using HTTP clients such as Postman, Bruno, and Thunder Client, Thunder Client. Note that in version 1.0.0 of the API, authentication tokens are required for certain endpoints.

Note:

Use any port, but in this guide, we will use port 5000 for demonstration purposes.

No token generation or authentication security is implemented here, as we are only creating a user (registration) without login. Authentication and token generation will be covered in a later versions.

Overview

The API includes endpoints for managing products, comments, and users. Here's how you can test each endpoint:

1. Products Endpoints

Get All Products

Method: GET

Endpoint: /api/products

Description: Retrieves a list of all products.

Authentication: Not required.

Testing in Postman:

  • Set method to GET
  • Enter URL: http://localhost:5000/api/products
  • Click Send

Get Product By ID

Method: GET

Endpoint: /api/products/:id

Description: Retrieves a single product by its ID.

Authentication: Not required.

Testing in Postman:

  • Set method to GET
  • Enter URL: http://localhost:5000/api/products/{id} (Replace {id} with the actual product ID)
  • Click Send

Create Product

Method: POST

Endpoint: /api/products

Description: Creates a new product.

Authentication: Not required.

Body:


{
  "name": "Product Name",
  "description": "Product Description",
  "price": 100,
  "category": "Category",
  "tags": ["tag1", "tag2"],
  "brand": "Brand",
  "stock": 10
}

Testing in Postman:

  • Set method to POST
  • Enter URL: http://localhost:5000/api/products
  • Select Body tab
  • Choose raw and set format to JSON
  • Enter the JSON payload above
  • Click Send

Update Product

Method: PATCH

Endpoint: /api/products/:id

Description: Updates an existing product by its ID.

Authentication: Not required.

Body: JSON object with fields to update.

Testing in Postman:

  • Set method to PATCH
  • Enter URL: http://localhost:5000/api/products/{id} (Replace {id} with the actual product ID)
  • Select Body tab
  • Choose raw and set format to JSON
  • Enter the JSON payload with fields to update
  • Click Send

Delete Product

Method: DELETE

Endpoint: /api/products/:id

Description: Deletes a product by its ID.

Authentication: Not required.

Testing in Postman:

  • Set method to DELETE
  • Enter URL: http://localhost:5000/api/products/{id} (Replace {id} with the actual product ID)
  • Click Send

Toggle Like Product

Method: POST

Endpoint: /api/products/:productId/like

Description: Toggles a like on a product for the current user.

Authentication: Token required.

Body:


{
  "user": "user_id"
}

Testing in Postman:

  • Set method to POST
  • Enter URL: http://localhost:5000/api/products/{productId}/like (Replace {productId} with the actual product ID)
  • Select Body tab
  • Choose raw and set format to JSON
  • Enter the JSON payload above
  • Add Authorization header with Bearer YOUR_TOKEN
  • Click Send
