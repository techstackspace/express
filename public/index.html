Express API Testing Guide for Version 1.16.0

This guide provides instructions on how to test various API endpoints using HTTP clients such as Postman, Bruno, Thunder Client, RapidAPI Client, and Indomnia.

Note:

Use any port, but in this guide, we will use port 5000 for demonstration purposes.

No token generation or authentication security is implemented here, as we are only creating a user (registration) without login. Authentication and token generation will be covered in a later versions.

Overview

The API includes endpoints for managing products, comments, and users. Here's how you can test each endpoint:

1. Products Endpoints

Get All Products

Method: GET

Endpoint: /api/products

Description: Retrieves a list of all products.

Authentication: Required.

Testing in Postman:

  • Set method to GET
  • Enter URL: http://localhost:5000/api/products
  • Click Send

Example Request:

GET /api/products?search=tablet&minPrice=200&maxPrice=800&category=electronics&sortBy=price&order=desc&page=2&limit=10

You can also get products by user id

Example Request:

GET /api/users/:userId/products?page=1&limit=5&minPrice=100&maxPrice=500&category=clothing&sortBy=price&order=asc

Get All Products by User

Method: GET

Endpoint: /api/products/user/:userId

Description: Retrieves a list of all products created by a specific user.

Authentication: Required.

Testing in Postman:

  • Set method to GET
  • Enter URL: http://localhost:5000/api/products/user/{userId} (Replace {userId} with the actual user ID)
  • Also you can enter URL: http://localhost:5000/api/users/{userId}/product (Replace {userId} with the actual user ID)
  • Click Send

Get Product By ID

Method: GET

Endpoint: /api/products/:id

Description: Retrieves a single product by its ID.

Authentication: Required.

Testing in Postman:

  • Set method to GET
  • Enter URL: http://localhost:5000/api/products/{id} (Replace {id} with the actual product ID)
  • Click Send

Create Product

Method: POST

Endpoint: /api/products

Description: Creates a new product.

Authentication: Required.

Body:


{
  "name": "Product Name",
  "description": "Product Description",
  "price": 100,
  "category": "Category",
  "tags": ["tag1", "tag2"],
  "brand": "Brand",
  "stock": 10,
  "rating": 4.5,
  "reviews": [],
  "likes": [],
  "comments": [],
  "user": "user_id",
  "images": [],
  "videos": [],
  "pdfs": []
}

Note: If uploading media files (e.g., images, videos, PDFs), include them under the key mediaFiles in the form data. The backend will automatically assign these files to the appropriate fields (images, videos, pdfs) based on their MIME types.

Testing in Postman:

  • Set method to POST
  • Enter URL: http://localhost:5000/api/products
  • Select Body tab
  • Choose form-data
  • Add the necessary text fields like name, description, etc.
  • Add a new key mediaFiles and upload files (images, videos, PDFs).
  • Click Send

Update Product

Method: PATCH

Endpoint: /api/products/:id

Description: Updates an existing product by its ID.

Authentication: Required.

Body: JSON object with fields to update. If uploading media files, include them under the key mediaFiles in the form data.

Testing in Postman:

  • Set method to PATCH
  • Enter URL: http://localhost:5000/api/products/{id} (Replace {id} with the actual product ID)
  • Select Body tab
  • Choose form-data
  • Add the fields to update.
  • Add a new key mediaFiles and upload files (images, videos, PDFs) if needed.
  • Click Send

Delete Product

Method: DELETE

Endpoint: /api/products/:id

Description: Deletes a product by its ID.

Authentication: Required.

Testing in Postman:

  • Set method to DELETE
  • Enter URL: http://localhost:5000/api/products/{id} (Replace {id} with the actual product ID)
  • Click Send

Toggle Like Product

Method: POST

Endpoint: /api/products/:productId/like

Description: Toggles a like on a product for the current user.

Authentication: Required.

Body:


{
  "user": "user_id"

Testing in Postman:
